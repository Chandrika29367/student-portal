# CampusERP - Professional Student Management Portal (MERN Stack)

CampusERP is a production-quality, responsive university ERP portal built using the MERN Stack. It offers secure authentication, dashboards for Administrators and Students, notices bulletin boards, grade sheets, attendance tracking, and billing fee management. It comes with a database seed script that populates 300 sample students, 20 courses, mock grades, and registrations.

---

## Tech Stack
* **Frontend**: React + Vite, Tailwind CSS, Axios, React Router v6, React Hook Form, Recharts, Lucide Icons
* **Backend**: Node.js, Express.js, JWT Authentication, bcryptjs, Morgan logging, CORS
* **Database**: MongoDB & Mongoose (Atlas Ready)

## Getting Started

### Prerequisites
Make sure you have [Node.js](https://nodejs.org/) installed (v18+ recommended).

### 1. Backend Setup
1. Open a terminal and navigate to the backend directory:
   ```bash
   cd backend
   ```
2. Install the backend dependencies:
   ```bash
   npm install
   ```
3. Configure the environment variables in `backend/.env`. Change `MONGO_URI` to your MongoDB Atlas connection string if desired:
   ```env
   PORT=5000
   MONGO_URI=mongodb://127.0.0.1:27017/student_portal
   JWT_SECRET=supersecretjwtkey12345!
   NODE_ENV=development
   ```
4. Run the seed script to populate 300 students, 20 courses, mock grades, and registrations:
   ```bash
   npm run seed
   ```
5. Start the backend developer server:
   ```bash
   npm run dev
   ```

### 2. Frontend Setup
1. Open a second terminal window and navigate to the frontend directory:
   ```bash
   cd frontend
   ```
2. Install the frontend dependencies:
   ```bash
   npm install --legacy-peer-deps
   ```
3. Start the Vite React development server:
   ```bash
   npm run dev
   ```
4. Open your browser to the local URL (usually `http://localhost:5173`).

## Features Walkthrough

### 🔑 Authentication & Security
* Hashed passwords using `bcryptjs` on user registration and change-password operations.
* Secured REST routes using custom JWT headers verifying roles.

### 📊 Administrative Control Dashboard
* **Metrics & Analytics**: Total students, courses, average student CGPA, and billing.
* **Charts**: Grade distribution bar chart, department-wise student distribution donut chart.
* **Bulletins**: Publish and delete announcements targeting specific semesters or departments.
* **Student Directory**: Full CRUD. Search, sort, and filter students by semester, department, or billing status.
* **CSV Exporter**: Click a button to instantly download the current filtered list as a CSV file.
* **Academics & Class Attendance**: Record class roll calls in bulk, log SGPA, and post semester grades.
* **Fees Billing**: Invoice new semester charges, toggle pending invoice status to paid, and track transaction IDs.

### 🎓 Student Dashboard
* **Curriculum**: View active semester enrolled courses. Self-register for courses offered in their department/semester.
* **Grade Sheet**: Check semester results history and cumulative CGPA.
* **Attendance Ring**: Review overall class attendance percentage and detailed daily logs.
* **Bills & Invoices**: Monitor paid/pending semester dues.
* **Profile Settings**: View enrollment cards, update telephone numbers, addresses, emergency parent contacts, and change passwords.
